### Bug #1: test_decompose_task_basic - AttributeError with ainvoke
**Location**: `backend/tests/agents/test_orchestrator.py::test_decompose_task_basic`

**Issue**: 
- Attempting to use `patch.object(orchestrator.llm, 'ainvoke', ...)` on a Pydantic model (ChatOpenAI)
- Pydantic's custom `__delattr__` prevented method patching
- Error: `AttributeError: 'ChatOpenAI' object has no attribute 'ainvoke'`

**Root Cause**:
- ChatOpenAI is a Pydantic BaseModel, which has special attribute handling
- Cannot directly patch methods on Pydantic models with `patch.object`

**Solution**:
- Changed from patching the method to replacing the entire LLM object
- Used `patch.object(orchestrator, 'llm', mock_llm)` instead
- Created AsyncMock with proper `ainvoke` method setup

**Code Changes**:
```python
# Before (BROKEN):
with patch.object(orchestrator.llm, 'ainvoke', new_callable=AsyncMock) as mock_llm:
    mock_llm.return_value = mock_response
    
# After (FIXED):
mock_llm = AsyncMock()
mock_llm.ainvoke.return_value = mock_response
with patch.object(orchestrator, 'llm', mock_llm):
```

### Bug #2: Google Docs API Test Failures - Mock Call Count Mismatch
**Location**: `backend/tests/agents/test_docs_agent_advanced.py::TestGoogleDocsAPIAdvanced`

**Affected Tests**:
- `test_apply_formatting`
- `test_apply_named_style`
- `test_insert_image`
- `test_insert_page_break`
- `test_create_bullet_list`

**Issue**:
- MagicMock chain `mock_service.documents().batchUpdate().execute.return_value` 
- Created multiple unwanted calls during chain traversal
- Assertion expected 1 call, but got 2 (initial chain + actual call)

**Root Cause**:
- Automatic return value creation in MagicMock chains
- Each `.` in the chain creates a new mock that gets counted as a call

**Solution**:
- Create explicit mock for the final chain element
- Set up proper return value without intermediate chain calls

**Code Changes**:
```python
# Before (BROKEN):
mock_service.documents().batchUpdate().execute.return_value = {"success": True}
result = api.apply_formatting(...)
mock_service.documents().batchUpdate.assert_called_once()

# After (FIXED):
mock_batch_update = MagicMock()
mock_batch_update.execute.return_value = {"success": True}
mock_service.documents().batchUpdate.return_value = mock_batch_update
result = api.apply_formatting(...)
mock_service.documents().batchUpdate.assert_called_once()
```

### Bug #3: test_init_basic - ResearchAgent Attribute Error
**Location**: `backend/tests/agents/test_research_agent.py::TestResearchAgentInitialization`

**Issue**:
- Test checked for `agent.citations` attribute which doesn't exist
- Test expected `ConversationMemory` but agent uses `MemoryManager`

**Root Cause**:
- ResearchAgent has `citation_tracker` and `get_citations()` method, not `citations` attribute
- BaseAgent was updated to use MemoryManager (Phase 2), tests not updated

**Solution**:
- Changed `agent.citations` to `agent.get_citations()`
- Updated type check from `ConversationMemory` to `MemoryManager`

**Code Changes**:
```python
# Before (BROKEN):
assert agent.citations == []
assert isinstance(agent.memory, ConversationMemory)

# After (FIXED):
assert agent.get_citations() == []
assert isinstance(agent.memory, MemoryManager)
```
